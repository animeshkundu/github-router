Quickstart

Authenticate once with GitHub Copilot, then start the proxy. Works on macOS, Linux, and Windows (WSL).

1) Authenticate (one-time)

npx github-router@latest auth

Stores your Copilot token locally at ~/.local/share/github-router/github_token.

2) Start the proxy

npx github-router@latest start --claude-code

Generates Claude Code launch commands automatically. Use --codex for Codex CLI.

3) Point your client

curl http://localhost:8787/v1/chat/completions \\  \\
  -H "Content-Type: application/json" \\  \\
  -d '{"model":"gpt-4.1","messages":[{"role":"user","content":"Hello"}]}'

OpenAI-compatible tools just need OPENAI_BASE_URL=http://localhost:8787/v1 and a dummy API key.

Questions

FAQ

Where are tokens stored?

By default in ~/.local/share/github-router/github_token. Pass --github-token to skip auth or mount a volume when using Docker.

Does streaming work?

Yes. Chat Completions, Responses, and Anthropic Messages stream over SSE with automatic end-of-stream detection.

Which models map to which endpoints?

Use /responses for codex models like gpt5.2-codex. Chat models (gpt-4.1, gpt-4o, claude-sonnet-4) work via /chat/completions or /messages depending on format.

Can I run it in Docker?

Yes. docker run -p 8787:8787 -e GH_TOKEN=your_token ghcr.io/animeshkundu/github-router. Mount a volume to persist tokens.

Is this official?

No. This is an unofficial proxy. Review GitHub Copilot terms and acceptable use policies before relying on it.
